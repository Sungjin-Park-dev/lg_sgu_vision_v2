### select_ik_dp.py — wrist_3 고정 시점 이동으로 CSV 재생 속도 균일화

**증상**: `trajectory_dp_*.csv` 재생 시 motion-planned (Phase 4 transit) 구간과 linear-interp 구간 사이의 joint-space 속도 차이가 시각적으로 보임.

**원인**: `interpolate_and_resample`은 6-DoF L2 cumulative arc-length로 균일 resample하지만, 그 *직후* `final_traj[:, -1] = wrist3_fixed`로 wrist_3 컬럼만 덮어쓰는 코드가 있었음 (line 1031-1033). 인접 row diff의 6번째 컴포넌트 d₆ 만큼 ‖·‖₂가 √(spacing² − d₆²) 로 줄어드는데, MotionGen transit은 wrist_3를 부드럽게 회전시키므로 d₆가 크고, linear interp 구간은 d₆가 작음 → transit 구간만 selectively 짧아져 "느려 보이는" 비균일.

**수정** (`scripts/pipeline/select_ik_dp.py`):
1. wrist_3 잠금을 Phase 3 종료 직후로 이동 (`selected[:, -1] = wrist3_fixed`). Phase 4 MotionGen이 동일 wrist_3 start/goal로 plan → 흔들림 최소화.
2. transit_segments 결과에도 wrist_3 강제 (안전망).
3. resample 후 wrist_3 override 제거.
4. 충돌 drop 후 cumulative L2 arc-length 재resample 추가 (gap으로 인한 jump 방지).

**결과** (sample/124, spacing=0.1):
- wrist_3 컬럼 모든 row = 0.0° (max deviation 0).
- L2 diff: mean=0.0909, 117/162 (72%) 가 [0.099, 0.100]. transit/non-transit 구간 차별 없음.
- 잔여 outlier 14% (<0.08): MotionGen transit이 backtrack한 구간(예: idx 139↔141 거의 같은 자세 복귀)에서 chord ≪ arc-length. 본 fix 범위 외.
- Reconfig 통계: 14 reconfigs (inter=7, intra=0; 이전 8 → 0 개선).
- Transit 7/7 성공, 충돌 0.

**수학적 등가성**: 입력 단계에서 wrist_3를 상수로 잠그면 6-DoF L2 = 5-DoF L2가 되어 "wrist_3를 metric에서 제외"하는 framing과 동일.
