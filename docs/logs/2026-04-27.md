### publish_trajectory.py — CSV joint 이름 prefix 자동 매칭

**증상**: `KeyError: 'shoulder_pan_joint'`. 다른 시뮬레이터용 CSV에는 `ur20_shoulder_pan_joint`처럼 prefix가 붙어 있음.

**해결** (scripts/pipeline/publish_trajectory.py:42-66): suffix 기반 자동 매칭. CSV 헤더에서 표준 joint 이름으로 *끝나는* 컬럼을 찾아 매핑. prefix 종류(`ur20_`, `ur5_`, 무접두) 무관. 매칭이 0개나 2개 이상이면 ValueError.

ROS publish 시에는 prefix 없는 표준 이름으로 보냄 (UR controller가 기대하는 형식). **두 종류 CSV를 만들 필요 없음** — 단일 소스.

### move_to_start.py — `Executor is already spinning` 버그 수정

**증상**:
```
RuntimeError: Executor is already spinning
File ".../move_to_start.py", line 77, in _on_startup
  rclpy.spin_once(self, timeout_sec=0.5)
```

**원인**: `_on_startup`은 timer 콜백 (executor가 이미 spin 중). 콜백 안에서 `rclpy.spin_once()` 재호출은 금지.

**수정**: timer 콜백에서 일찍 return하고 다음 tick에서 재시도하는 패턴 (publish_trajectory.py와 동일). joint_states 도착 후에만 timer cancel + 진행.

추가로 보수적 기본값: `--duration` 3.0→5.0s, `--max-vel` 1.0→0.5 rad/s. 178° 회전 시 controller의 `Position Tolerance: 0.200000` 위반(error_code=-4) 자주 발생해 abort되던 문제 해결.

### Isaac Sim USD — GUI URDF Importer로 전환

**증상**: `urdf_usd_converter`로 변환한 USD에서 `Available DOFs: []` 경고. PhysX가 articulation root는 `/World/UR20/Geometry/world/base_link`로 인식하지만, joint들은 `/World/UR20/Physics/`에 있어 ArticulationRoot의 하위 트리에 없음 → DOF 0개.

**원인**: `urdf_usd_converter`의 Atomic Component layout이 geometry/physics를 별도 sub-namespace로 분리. ArticulationRoot가 둘 다 포함하는 공통 부모에 있어야 함.

**시도한 우회**: 런타임에 `RemoveAPI` + `Apply` 로 ArticulationRoot 위치 이동 — 동작은 하나 매번 실행해야 하는 임시방편.

**해결**: Isaac Sim의 GUI URDF Importer 사용. articulation root 위치 등 변환 옵션을 명시적으로 컨트롤 가능. 결과물 `ur20_description/ur20/ur20.usd`로 저장.

### URSim 셋업 + FastCDR ABI 충돌 수정

**증상**:
```
libpal_statistics_msgs__rosidl_typesupport_fastrtps_cpp.so:
undefined symbol: _ZN8eprosima7fastcdr3Cdr9serializeEj
```

`controller_manager`가 startup 직후 죽음 → spawner들이 service timeout으로 줄줄이 실패.

**원인**: `pal_statistics_msgs`가 새 FastCDR ABI에 빌드됐지만 컨테이너의 `libfastcdr`는 옛 버전. apt 인덱스 stale.

**해결**: `apt update && apt upgrade -y`. fastdds/fastcdr/pal_statistics가 일관된 버전으로 정렬됨.

URSim 시작 (UR20):
```bash
ros2 run ur_client_library start_ursim.sh -m UR20
ros2 launch ur_robot_driver ur_control.launch.py \
    ur_type:=ur20 robot_ip:=192.168.56.101 launch_rviz:=true
```

External Control URCap의 Host IP는 호스트의 vboxnet 인터페이스(`192.168.56.1`)로 설정 — URSim과 같은 서브넷.

### .venv/bin/activate — Isaac Sim ROS2 bridge env 추가

Isaac Sim이 자기 시뮬레이션에서 ROS2 bridge를 띄울 때 사용하는 env 변수 셋. activate 시 `LD_LIBRARY_PATH`에 isaacsim 번들 jazzy lib 추가, `ROS_DISTRO`/`RMW_IMPLEMENTATION` 설정. deactivate 시 원래 값 복원.

단, **시스템 ROS2(`/opt/ros/jazzy`)와 동시에 sourcing 시 ABI 충돌 위험**. 다른 셸에서 운용 권장:
- 셸 A (driver/RViz): `source /opt/ros/jazzy/setup.bash`만, venv 비활성
- 셸 B (Isaac Sim): venv 활성, 시스템 ROS2 sourcing X
- 셸 C (publish/move 스크립트): 시스템 ROS2 + venv (rclpy는 시스템 우선 사용 위해 ROS2 sourcing 우선)
