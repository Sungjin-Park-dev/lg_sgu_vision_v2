### select_ik_dp.py — DBSCAN + DP 기반 IK 해 선택 모듈 신규 작성

- `scripts/pipeline/select_ik_dp.py` 신규 생성
- 기존 `plan_motion.py`의 greedy IK 선택(seed 4개, 직전 해에서 가장 가까운 후보)을 대체하는 전역 최적화 모듈

#### 알고리즘 3단계
1. **Phase 1 — Multi-seed IK**: viewpoint당 100개 seed로 IK 풀이 (`solve_batch`, batch_size=4)
2. **Phase 2 — DBSCAN**: viewpoint당 성공한 IK 해를 joint-space에서 클러스터링 → 대표 해(medoid) 추출 (eps=0.3 rad, avg ~3.9개/viewpoint)
3. **Phase 3 — DP**: 대표 해 간 최소 비용 경로 선택
   - 비용: `is_reconfig × 1000 + L2_distance` (1순위: reconfig 최소화, 2순위: joint distance)
   - reconfig 판정: angular L-inf > threshold (기본 29°)

#### Joint angle wrapping 적용
- 문제: IK 해 간 raw diff가 ~360°/~180°로 계산되어 가짜 reconfig 다수 발생 (28/106)
- 원인: revolute joint에서 -270°와 +90°가 물리적으로 같은 자세이나 `|(-270)-90|=360°`로 계산
- 해결: `angular_diff(a, b)` 함수 도입 — `min(d, 2π-d)` wrapping 적용
- 적용 위치: DBSCAN metric, DP 비용, reconfig 분석, 시각화 판정 (4곳)
- 결과: intra-cluster reconfig 28→8, max jump 429°→147°, mean jump 68.7°→13.3°

#### 클러스터 간/내 reconfig 분석 기능
- `cluster_id` 기반으로 inter-cluster vs intra-cluster reconfig 구분 출력
- intra-cluster reconfig 발생 시 개별 viewpoint 인덱스와 jump 크기 로그

- `trajectory_dp_anim.html` — animated: 슬라이더/Play/Pause로 step별 경로 성장

#### Joint angle 정규화 추가
- angular_diff wrapping 방식에서 **IK 해 정규화 방식으로 전환**
- 문제: angular_diff가 2π 차이를 0으로 계산 → DP가 oscillation을 허용 (CSV에서 shoulder_pan이 -4.29 ↔ 1.98 반복)
- 해결: `normalize_joints(q) = ((q + π) % 2π) - π` 를 Phase 1 직후 적용
  - -4.29 rad → 1.99 rad, 1.98 rad → 1.98 rad (같은 값으로 통일)
- angular_diff/angular_distance_metric 제거, DBSCAN/DP 모두 일반 euclidean으로 복원
- CSV 헤더에 robot name prefix 추가 (`ur20-shoulder_pan_joint` 등)
- `--viewpoints` 인자 추가 (직접 h5 파일 경로 지정 가능)

#### 실행 예시 결과 (sample, 124 viewpoints)
