## 한 일

### 곡면 커버리지 — 평면 그리드 → 표면 직접 균일 샘플링

**문제**: 기존 `generate_grid_viewpoints`는 PCA 평면에 균일 그리드를 깔고 closest-point로 표면에
스냅. ① 점 예산을 **투영 면적**으로 잡고(`n_rows/n_cols`), ② PCA 평면에 수직인 곡면·측벽은
투영에서 얇은 띠가 되어 closest-point가 평면-나란한 윗면으로만 쏠림 → **곡면에 점이 거의 안 생김**.

**해결**: `generate_surface_viewpoints` 추가 — `trimesh.sample.sample_surface_even`(Poisson-disk).
- 핵심: `radius` 직접 지정은 under-pack(saturation ~200개에서 멈춤). **target count 방식**으로,
  원하는 표면 간격 `s`에 대해 `count = area / (2·s²)`로 역산(trimesh 내부 radius=√(area/2count)).
- `ViewpointGenParams.sampling_mode = 'grid' | 'surface'`, `prepare_grid` 분기. bottom 필터 유지.
- 검증(커버리지=표면 20k 샘플의 최근접 viewpoint 거리, max gap):
  cylinder 33→16mm, square **68.5→21.2mm**, curved 개선. grid가 비우던 곡면을 표면이 균일하게 덮음.

### 순서 — 전역 PCA-zigzag → surface-intrinsic 그래프

**문제**: `reorder_zigzag`/`compute_cluster_internal_order`가 전역 PCA 축에 의존 → 입체물에서 의미 약함.

**해결**: `order_cluster_graph` — 클러스터 **평균 법선**으로 탄젠트 2D 프레임(PCA 아님) →
`scipy.spatial.Delaunay`(degenerate면 kNN/tangent-정렬 폴백) → NN seed + open-path 2-opt.
- `ordering_mode = 'zigzag' | 'graph'`. `compute_cluster_internal_order`에 분기(graph는 permutation
  직접 반환 — argsort 안 함). 반환 dict는 zigzag와 동일 → GTSP 단계 무변경.
- **anti-explosion 가드**: 결과가 tangent-정렬 baseline보다 길면 폴백(zigzag보다 절대 악화 없음).
- 평면 10×10 테스트에서 최적 serpentine 길이(99.0) 달성. n≤2/공선/중복점/NaN 안전.

### 밀도·eps 조절 (스튜디오)

- **surface spacing 슬라이더**(5–40mm, 기본 **12mm**≈60% 오버랩). `count≈area/(2·spacing²)`라 제곱
  반비례 — curved 기준 9→374, 12→206, 20→80개. 캐시 키에 spacing 포함.
- **eps 자동 추적**: surface 모드에서 spacing 바꾸면 eps가 `1.5×spacing`으로 자동(슬라이더는 유지 →
  수동 조절 가능, 다음 spacing 변경 시 리셋). grid 모드는 추적 안 함. (`EPS_SPACING_FACTOR`)

### 클러스터링 — 밀도법 → 공간 분할(Agglomerative) + max_span

**문제**: surface 샘플링은 밀도가 **균일**해서 DBSCAN(밀도 기반)이 자를 골짜기가 없음 →
"거대 클러스터 1개 + 싱글톤 다발"(curved 206vp → 36클러스터 중 **18 싱글톤**). 균일 표면 클러스터링은
밀도법(DBSCAN/HDBSCAN)이 아니라 **공간 분할**이 맞음.

**해결**: `cluster_agglomerative`(sklearn, 의존성 0) + `cluster_coacd_agglomerative`,
method `agglomerative`/`coacd+agglomerative`.
- 노브 = **`max span (mm)`**: `AgglomerativeClustering(distance_threshold, linkage='complete')` →
  **클러스터 지름 ≤ 값 보장**(멀리 떨어진 점이 한 클러스터로 묶임 방지). single linkage는 체이닝으로
  지름 폭증 → 반드시 complete. (대안 `--target-size`: Ward+개수, 지름 무제한.)
- 결과: 싱글톤 0, 지름 보장(span=60→실측 지름≤60). 스튜디오 Sub-cluster 드롭다운으로 dbscan과
  나란히 비교(CoACD 분해는 공유 캐시).

### 카메라 공간 클러스터링 + 색 재사용 수정 (사용자 버그 제보)

**문제**: max_span을 줄여도 "멀리 떨어진 viewpoint가 한 클러스터"로 보임. 원인 둘:
1. coacd 2단계가 **표면 위치**로 클러스터링했는데, 렌더 마커·로봇 EE는 **카메라 위치**(표면+법선×110mm).
   곡면에선 표면 60mm가 카메라 **225mm**로 벌어짐 → max_span이 엉뚱한 공간을 제한.
2. 색 팔레트 25색 순환 → 클러스터>25면 멀리 떨어진 두 클러스터가 같은 색 → "한 클러스터처럼" 보임
   (max_span 줄이면 클러스터↑ → 오히려 악화).

**해결**:
1. `cluster_coacd_dbscan`/`cluster_coacd_agglomerative`에 `camera_positions` 인자 추가, 2단계를
   카메라 위치 기준으로. → max_span이 **마커/EE 지름**을 직접 제한(span=60→카메라 지름≤60 검증).
2. `distinct_colors(K)`(황금비 HSV)로 K개 **고유 색** 생성(44개도 44/44 고유). `_build_scene`에서 사용.
