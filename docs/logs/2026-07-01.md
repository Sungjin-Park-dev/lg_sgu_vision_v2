## 1. ★ cylinder_sample GLNS "0 reachable" — cuRobo 소형 mesh 충돌 오판

**증상**: `solve_glns_path.py` Phase-1 IK `0/19712 (0.0%)` → `No reachable viewpoints remain`. curved_structure
/sample 은 같은 코드로 정상.

**진단 (scratchpad 격리 실험)**:
- 빈 world(kinematic only) → **250/704 (35.5%)** 성공, top-down vp 도 11~17 해 → **reachability 문제 아님**.
- full collision world → **0/704**. 부위별로 쪼개니 table/support/wall 아니라 **target mesh 자체와 100% 충돌**.
- 결정적 증거: 물체에서 0.5m 떨어진 **home 자세**조차 이 mesh 와 충돌 → **blanket false positive**.
- 원인은 **크기**(모양·면수 무관): 최소 bbox 치수 ≲ 5cm 인 메시는 cuRobo mesh 충돌이 world 전체를 충돌로
  오판. curved(3022f)·box(0.30×…)=정상, primitive cylinder(128f)·sphere·**curved 를 ×0.2 축소** = 오판.
  WD 46→200mm 로 늘려도 무효(카메라 거리 문제 아님).

| 충돌 프록시 | 최소 치수(m) | home 충돌? |
|---|---|---|
| curved 실메시 | 0.06 | 0 ✓ |
| box 큰거(0.30×0.148×0.06) | 0.06 | 0 ✓ |
| **cylinder_sample(36432f)** | **0.046** | **1 ✗** |
| primitive cylinder 128f | 0.046 | 1 ✗ |
| curved ×0.2 축소 | 0.012 | 1 ✗ |

**수정**: `build_collision_world()` 에서 등록 물체는 mesh 대신 **mesh bbox 를 Cuboid(obb)** 로. Cuboid 는
analytic 이라 모든 충돌 consumer(IK·transit·verify)에서 확실히 반영(env cuboid 들이 250→137 실제 컷 확인).
dims/center 는 mesh bbox 에서 자동 산출(rotation 반영) → 메시 바뀌어도 추적.

```python
# config.py
OBJECT_COLLISION_SHAPE = {"cylinder_sample": "box"}
```

**검증**: 패치된 `build_collision_world('cylinder_sample')` → home 충돌 0, full IK **137/704, reachable 21/22**
(남은 1개 vp17 은 kinematic 으로도 0해 = 진짜 unreachable). curobo.scene 에 Cylinder primitive 도 있으나
`collision_cache={"obb","mesh"}` 에 cylinder 키 없어 무시 위험 + 내부 mesh 변환 시 동일 버그 재발 우려 →
**box 채택**. Ø46 물체는 46mm standoff 라 물체가 팔 충돌에 사실상 영향 없어 bbox 과대근사도 저위험.

## 2. ★ square_structure viewpoint 가 "물체 안"에 생성 — 속 빈 물체 안쪽 면 샘플링

**증상**: viewpoint_studio 로 생성하면 viewpoint 가 물체 내부에 생김.

**진단**: mesh 는 watertight/winding OK 이나 **volume 이 bbox 의 4%**, surface area 는 외피의 1.75×,
법선이 centroid 향하는 face ≈49% → **속 빈 얇은 벽 상자**. `m.section()` 단면으로 확정(U자 = open box).
표면 샘플링(`trimesh.sample.sample_surface`)이 **안쪽 면까지 뽑고** 그 법선이 공동 안을 향해
`camera = pos + normal·wd` 가 상자 내부로 들어감(메시·법선 자체는 정상).

**수정**: 생성 후 **바깥 껍데기만** 남기는 필터를 물체별 opt-in 추가. 각 표면점 법선 vs **최근접
convex-hull 면의 바깥 법선** dot < 임계면 안쪽 껍데기로 보고 제거(바깥=align≈+1, 안쪽=align≈−1로 이분).
`hull.nearest.on_surface(positions)` → `hull.face_normals[tri]`.

```python
# config.py
OBJECT_FILTER_INTERIOR = {"square_structure": {"hull_align_min": 0.3}}
# generate_viewpoints.py: filter_interior_viewpoints(mesh, positions, normals, hull_align_min)
#   ViewpointGenParams.filter_interior / interior_hull_align_min ; prepare_grid bottom-filter 뒤 적용
#   viewpoint_studio._generate_worker & CLI main 이 config lookup 으로 켠다
```

**검증**: square_structure **117 → 71** vp(안쪽 껍데기 46 = 안쪽 바닥+안쪽 벽 전부 제거). curved_structure
100 그대로(미등록 → 미적용). 볼록 물체는 align≈1 이라 영향 0.

## 시도 → 폐기

- **exterior-visibility(escape-ray) 필터** (§2): 카메라 바깥 반구로 광선 쏴 안 빠져나가면 제거. 얕고 넓은
  tray 는 안쪽 바닥이 열린 위로 보여 escape≈1 → **12개(완전히 갇힌 것)만** 제거, "위에서 내려다보는 안쪽
  바닥"은 못 지움. "바깥 껍데기만" 요구엔 부족 → **hull-normal 방식으로 교체**(가시성 아닌 껍데기 구조 판정).
- **§1 에서 WD 늘리기 / Cylinder·Capsule primitive**: WD 무효, primitive 는 cache/내부변환 위험 → Cuboid.

## 주의 / 남은 것

- `OBJECT_FILTER_INTERIOR` hull-normal 필터는 **바깥 형상이 오목한**(홈·계단) 물체엔 그 면도 지울 수 있음 →
  box 류에만 opt-in. 세기는 `hull_align_min`(0.3→0.5 로 더 엄격) 조정.
- `cylinder_sample` 배치는 현재 CLI `--object-position -0.1 0.8 0.19`(curved 자리) override. 굳히려면
  `OBJECT_PLACEMENTS` 에 항목 추가.
- 두 수정 모두 물체별 opt-in 표(`OBJECT_COLLISION_SHAPE`, `OBJECT_FILTER_INTERIOR`)라 기존 물체 무영향.
