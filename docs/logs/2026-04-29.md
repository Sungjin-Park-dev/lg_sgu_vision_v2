### 좌표계 결정 — robot frame == world XY origin

두 옵션 비교 후 **robot frame 유지** 선택:
- (A) 채택: robot/mount를 world XY 원점에, environment USD를 +X로 shift → cuRobo/IK/기존 config 좌표 변환 불필요
- (B) 기각: robot을 world `(-2, 0, 0)`에 배치 → cuRobo world spec 매번 -2 빼야 함, 좌표 소스 분열

**결론**: 로보틱스 컨벤션대로 robot base_link = world XY origin. Z만 mount 높이만큼 (805mm) 들어 올림. environment.usd 는 `(+2, 0, 0)` 으로 평행이동.

### USD 정렬 — bbox 측정 후 위치/스케일 보정

각 USD의 **로컬 origin 위치 + intrinsic 치수**를 모르고 추측 불가 → bbox 헬퍼(`UsdGeom.BBoxCache.ComputeWorldBound`)로 직접 측정 후 정렬.

측정 결과:
| USD | origin 위치 | intrinsic 크기 | 목표 치수 | 보정 |
|---|---|---|---|---|
| `ur20.usd` | base 하단 | — | — | position.z = 0.805 |
| `ur10_mount.usd` | **상단** (max_z=0) | 0.515m | 0.805m | scale_z = 1.563, position.z = 0.805 |
| `thor_table.usd` | **상단** (max_z=0) | 0.795m | 0.630m | scale_z = 0.793, position.z = 0.630 |

추가 조정:
- Mount XY scale `2.0` (270×270mm → 540×540mm) — 시각적으로 더 자연스러운 두께
- Table USD origin은 X 방향 +0.270m 치우침 → position에서 보정값 빼서 정렬
- Target object USD 내부 scale=0.01 박혀 있음 → `prims.create_prim(scale=(1,1,1))` 로 덮어씀

최종 world bbox:
```
Mount:  z=[0.000, 0.805]  (XY 540×540mm)
Robot:  base @ z=0.805 (mount 윗면)
Table:  center (-0.2, 1.1), z=[0.000, 0.630]
Object: center (-0.1, 1.1, 0.795), 150×223×145mm (table top 위 165mm 띄움 — 의도적)
```

### thor_table.usd / ur10_mount.usd — vMaterials_2 머티리얼 교체

**증상**:
```
[Error] [omni.usd] USD_MDL: ... 'rtx::neuraylib::MdlModuleId' for
        '../../Materials/vMaterials_2/Metal/Aluminum_Brushed.mdl' is Invalid
[Error] [omni.rtx.materials] Unable to find SdrShaderNode for prim:
        '/World/Table/Looks/Aluminum_Brushed/Shader' ...
```

**원인**: USD가 NVIDIA vMaterials_2 라이브러리를 상대 경로 (`::..::..::Materials::vMaterials_2::...`)로 참조. USD 디렉토리에서 `../..` 올라간 자리에 `Materials/vMaterials_2/...` 가 있어야 하는데 시스템에 vMaterials_2 미설치 (확인 결과 `.venv` 안에 test 일부만 있음, Metal 모듈 없음).

**해결**: Isaac Sim GUI에서 USD 열고 vMaterials shader → OmniPBR로 교체 후 Save Flattened. 외부 다운로드/검색경로 설정 불필요, 일회성 작업.

## 결정/변경

- **로봇 base = world XY origin** — cuRobo/config 호환성 위해. Z만 mount 805mm 띄움.
- **Visual ↔ cuRobo 충돌 spec 분리 유지** — visual은 USD, cuRobo는 cuboid primitive. 좌표/치수만 동기화. cuRobo가 USD mesh collision 직접 읽는 것보다 빌드 시간/디버깅 측면에서 유리.
- **Workcell Xform 도입 검토 → 폐기** — 처음엔 robot+mount+table을 한 Xform 묶어 같이 이동시키려 했으나, robot=원점 결정 후 불필요. flat 구조 (`/World/UR20`, `/World/Mount`, `/World/Table`, `/World/Environment`).
- **Table USD origin 위치는 상단 (max_z=0)** — bbox 측정으로 확인. mount도 동일.
- **Mount XY scale 2배** — 사용자 결정 (시각적 자연스러움). Z는 805mm 유지.
